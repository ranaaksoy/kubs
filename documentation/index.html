Getting Started

Inside the project folder you'll find following folders and files.

  • /src - Folder with all source files
  • /public - Folder with static assets
  • .env.local - Environment variables
  • tailwind.config.js - Tailwind config
  • types.d.ts - TypeScript types

Usage

Make sure that Node JS is installed to run NPM scripts

Install the project dependencies

npm install

Once installed, run one of the following commands:

  1. npm run dev - Starts development script
  2. npm run start - Starts preview script
  3. npm run build - Starts production script
  4. npm run email:dev - Starts email development script

Site Config

Edit the site config file located in src/config/site.ts. This file contains SEO metadata as well as header and footer navigation links and WP_REST_BASE_URL constant required for the headless WordPress integration. Your WordPress base URL should have the following format: https://yoursite.com/wp-json/wp/v2

Edit your next.config.js file and include your WordPress domain there:

        
const nextConfig = {
  images: {
    remotePatterns: [
      {
        protocol: "https",
        hostname: "deothemes.com",
      },
      {
        protocol: "https",
        hostname: "your-wordpress-site.com",
      },
      {
        protocol: "https",
        hostname: "secure.gravatar.com",
      },
    ],
  },
};

module.exports = nextConfig;

Environment Variables

Edit .env.local file which includes environment variables for Resend, Mailchimp and WordPress

  1. NEXT_PUBLIC_APP_URL - Use your production URL when deploying to production
  2. RESEND_API_KEY - Resend API Key found at resend.com/api-keys
  3. EMAIL_FROM_ADDRESS - Register a domain at resend.com/domains. Email provided by resend only for testing: onboarding@resend.dev
  4. EMAIL_TO_ADDRESS - Your email address.
  5. MAILCHIMP_API_KEY - Mailchimp API key. Please refer to Mailchimp section for more details.
  6. MAILCHIMP_API_SERVER - Mailchimp API server. Please refer to Mailchimp section for more details.
  7. MAILCHIMP_AUDIENCE_ID - Mailchimp Audience ID. Please refer to Mailchimp section for more details.

Contact Form

Contact and Quotation forms are using Resend to send the emails. Create a free account and create an API key to get started. Copy the API key and paste it in .env.local as RESEND_API_KEY=your-key-here

Register a domain at resend.com/domains Or you can use this email provided by resend for only testing: onboarding@resend.dev. It is not recommended though

Edit the EMAIL_FROM_ADDRESS and EMAIL_TO_ADDRESS fields in .env.local. 

RESEND_API_KEY=your-key-here
EMAIL_FROM_ADDRESS="onboarding@resend.dev"
EMAIL_TO_ADDRESS="youremail@gmail.com"

Mailchimp Form

Create a free Mailchimp account to get started. To make your form functional you'll need to edit .env.local and paste your values.

Follow this article to create an API key.

To find your API server, login to your Mailchimp account and see the URL

To find your Audience ID follow this article

.env.local Code reference:

MAILCHIMP_API_KEY=your-api-key
MAILCHIMP_API_SERVER=your-api-server
MAILCHIMP_AUDIENCE_ID=your-audience-id

Google Map

To enable Google map you'll need to create and enter your API key. Follow this article for more details.

To find the Latitude and Longitude you can use this free tool - https://www.latlong.net/

Edit src/components/sections/section-google-map.tsx file and paste your key there.

WordPress Demo Import

Install the WordPress to get started. Make sure to install the ACF plugin to get the custom posts and fields for Projects.

Navigate to Tools > Import in your admin dashboard and Run the Importer. Select the file wordpress-demo-import.xml provided in downloadable archive and click Upload file and Import.

Deployment

To deploy your project to Vercel follow this article - Deploying to Vercel. Login to your Vercel account, connect your GitHub or Bitbucket repository. When you push the changes to your repository Vercel will automatically start the deployment process. Don't forget to add your Environment variables.
